# 機能追加リスト（実装しやすい管理版）

このドキュメントは、機能追加時に「何を」「どの順で」「どこまで作るか」を揃えるための管理表です。  
新しい機能は、下の**追加テンプレート**を使って同じ粒度で追記してください。

---

## 追加テンプレート（この形式で追記）

```md
## [機能名]
- 優先度: 高 / 中 / 低
- ステータス: 未着手 / 設計中 / 実装中 / レビュー中 / 完了
- 目的: （この機能で解決したい課題）
- 対象: Backend / Web / Discord Bot / DB
- 実装スコープ:
  - （必須要件1）
  - （必須要件2）
- 受け入れ条件:
  - （完了判定1）
  - （完了判定2）
- 備考:
  - （将来拡張や依存関係）
```

## チケットシステム導入
- 優先度: 高
- ステータス: 未着手
- 目的: Discord上で問い合わせ対応を整理し、運営対応の抜け漏れを防ぐ。
- 対象: Discord Bot / Backend / DB / Web
- 実装スコープ:
  - チケット作成（ボタンまたはコマンド）
  - チケット専用チャンネルの自動作成と権限制御
  - ステータス管理（Open / In Progress / Closed）
  - 担当者アサイン、対応履歴、クローズ時のログ保存
  - GiveawayのClaimボタンとの連携も（Claim Ticket）
- 受け入れ条件:
  - 一般ユーザーがチケットを作成できる
  - 運営がチケットの状態変更とクローズを実行できる
  - クローズ後に対応ログを確認できる
- 備考:
  - 将来的にFAQ自動応答や優先度分類と連携可能にする

## 監査ログ（操作履歴）
- 優先度: 高
- ステータス: 未着手
- 目的: 設定変更や抽選操作の追跡性を高め、トラブル調査をしやすくする。
- 対象: Backend / DB / Web
- 実装スコープ:
  - 設定変更、Giveaway作成・終了・再抽選をイベント保存
  - 実行者、対象サーバー、変更差分、実行時刻を記録
  - ダッシュボードで期間検索・絞り込み表示
- 受け入れ条件:
  - 主要操作のログが欠落せず保存される
  - 期間とイベント種別で検索できる
- 備考:
  - チケットシステム導入時の対応履歴保存にも流用可能

## 権限エラー理由表示の強化
- 優先度: 高
- ステータス: 未着手
- 目的: 「なぜ操作できないか」を利用者がすぐ理解できるようにする。
- 対象: Backend / Web / Discord Bot
- 実装スコープ:
  - 権限不足ケースごとのエラーメッセージ出し分け
  - 必要ロール不足、対象未設定、オーナー限定を明確化
  - 解決手順への導線表示
- 受け入れ条件:
  - 主要な権限失敗パターンで理由を表示できる
  - 画面や応答メッセージから修正手順が分かる
- 備考:
  - i18n対応を前提に文言キー設計する

## Giveawayテンプレート機能
- 優先度: 高
- ステータス: 未着手
- 目的: 定型イベント作成の手間と入力漏れを減らす。
- 対象: Backend / Web / DB
- 実装スコープ:
  - タイトル、説明、当選人数、期限、投稿先をテンプレート化
  - テンプレート作成・編集・複製・削除
  - サーバー単位で複数テンプレート管理
- 受け入れ条件:
  - テンプレート適用で作成画面の初期値が反映される
  - テンプレートCRUDが一通り実行できる
- 備考:
  - 将来の予約投稿機能と組み合わせる

## 重複作成防止（冪等性キー）
- 優先度: 高
- ステータス: 未着手
- 目的: 連打や再送による同一Giveaway重複作成を防ぐ。
- 対象: Backend / Web
- 実装スコープ:
  - リクエストごとの冪等性キー発行・検証
  - 同一キー再送時は既存結果を返す
  - UIで送信中状態を固定し多重送信を防止
- 受け入れ条件:
  - 同一操作の短時間連続送信で重複データが作られない
- 備考:
  - 失敗時リトライ設計と合わせて整理する

## 下書き保存機能
- 優先度: 中
- ステータス: 未着手
- 目的: 入力中断時の再入力負担を減らす。
- 対象: Web / Backend / DB

## フィルター/検索拡張
- 優先度: 中
- ステータス: 未着手
- 目的: 過去開催の確認速度を上げる。
- 対象: Web / Backend
- 実装スコープ:
  - 作成者、チャンネル、期間、ステータス、キーワード検索
  - 複合条件と並び替え
  - よく使う条件の保存
- 受け入れ条件:
  - 複数条件で期待通りに一覧が絞り込める

## 通知チャンネル設定
- 優先度: 中
- ステータス: 未着手
- 目的: 運営通知の見逃しを減らす。
- 対象: Backend / Web / Discord Bot
- 実装スコープ:
  - 終了通知、再抽選通知、期限切れ通知の送信先分離
  - 通知文言テンプレート
  - テスト送信
- 受け入れ条件:
  - 設定した通知先にイベント別通知が送信される
