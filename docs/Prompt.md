OAuth2でDiscordログインを追加し、ログインした人物がこのサーバーの管理者であることを確認してさまざまなことができるようにしてください。サーバー設定にダッシュボード閲覧可能ロールを追加し、複数のロールを選択できるようにしてください。そのロールを持ってる人と管理者はダッシュボードにログインできます。
一つのページにすべて表示するのではなく、ハンバーガーメニューにサーバー設定とGiveaway作成と進行中Giveawayの3つの項目を表示してそれらのどれかを押すことでそれを表示できるようにしてください。
Giveaway作成の実行ユーザーは要りません (上記のログイン部分があるためログインした人物が実行したことにする)。
Giveaway作成のチャンネルは設定で選択されたGiveaway作成可能チャンネルから選択するようにしてください。
react-datepicker
date-fns
chrono-node
これらを使ってGiveaway作成の終了日時を入力するテキスト欄に自然言語(chrono-node)(yyyy/mm/ddなど)で入力を始めるとカレンダーが表示されてカレンダーを選択しても日にちを入力できるようにしてほしいです。出てきた日にちはdate-fnsで整形してください。
Giveaway作成にトグルボタンとして自動作成するかどうかを決めるボタンを追加してください。それをオンにすると「終了日時」が「間隔」にかわります。これもさっきと同じようにカレンダーなどを表示しますが、カレンダーからのピックアップする機能は消してください。1h, 20m, 1w2d3hなどのparseはchrono-nodeだと弱いので自前parserを作ってください。
全ての設定はまずDBから取得してそれを表示するようにしてください。
WebとBackend両方に共通することですが、管理ロールから「Giveaway作成可能ロール」に名前を変えてください。
