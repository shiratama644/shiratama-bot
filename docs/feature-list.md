# 追加すべき機能一覧

このドキュメントは、現状の Dashboard / Giveaway 機能をベースに、次に追加すると価値が高い機能を整理した一覧です。

## 優先度: 高（早めに着手）

1. **監査ログ（操作履歴）**
   - 対象: 設定変更、Giveaway作成、終了、再抽選
   - 目的: 誰がいつ何を操作したかを追跡できるようにする

2. **権限エラー時の理由表示強化**
   - 対象: サーバー選択不可、設定保存不可、Giveaway作成不可
   - 目的: 「なぜ操作できないか」をUIで明示し、問い合わせを減らす

3. **Giveawayテンプレート機能**
   - 対象: タイトル、説明、当選人数、受け取り期限の保存
   - 目的: 毎回同じ入力を繰り返す手間を削減する

4. **下書き保存機能**
   - 対象: Giveaway作成フォーム
   - 目的: 入力途中で画面遷移しても内容を復元できるようにする

5. **失敗時リトライ/通知改善**
   - 対象: API失敗時の表示と再試行導線
   - 目的: 一時的な失敗で操作不能にならないようにする

## 優先度: 中（運用効率の改善）

6. **Giveaway複製機能**
   - 対象: 既存Giveawayをコピーして再作成
   - 目的: 類似キャンペーンを短時間で再実施できるようにする

7. **フィルター/検索の拡張**
   - 対象: 作成者、チャンネル、期間、ステータスの複合検索
   - 目的: 過去Giveawayを運用視点で探しやすくする

8. **参加・当選統計ダッシュボード**
   - 対象: 参加者数推移、当選分布、開催頻度
   - 目的: Giveaway運用の改善判断に使える指標を提供する

9. **通知チャンネル設定**
   - 対象: 終了通知、再抽選通知、受け取り期限切れ通知
   - 目的: 運営メンバーへの情報共有を自動化する

10. **ロール設定のプリセット**
    - 対象: 使用可能ロール / 作成可能ロールの初期セット
    - 目的: 新規導入時の設定コストを下げる

## 優先度: 低（将来的な拡張）

11. **多言語文言の強化（i18n拡張）**
    - 対象: エラーメッセージ、通知メッセージ、管理UI補助文言
    - 目的: 日本語/英語運用の品質を揃える

12. **Webhook連携**
    - 対象: 外部ツール（運営ダッシュボード等）へのイベント配信
    - 目的: Bot外の運用基盤と連携しやすくする

13. **CSVエクスポート**
    - 対象: Giveaway一覧、当選結果、操作履歴
    - 目的: レポート作成や監査対応を容易にする

14. **ヘルプ/オンボーディング導線**
    - 対象: 初回利用者向けのUIガイド
    - 目的: 初見でも設定～運用まで迷わず進められるようにする

---

## 実装順の提案（最短）

2. 権限エラー表示強化  
3. テンプレート + 下書き保存  
4. 複製機能 + 検索拡張  
5. 統計・通知・外部連携
