# 追加すべき機能一覧（詳細版）

このドキュメントは、現状の Dashboard / Giveaway 機能をベースに、次に追加すると価値が高い機能を **目的** と **内容（実現イメージ）** つきで整理した一覧です。

## 優先度: 高（早めに着手）

1. **監査ログ（操作履歴）**
   - **目的**: 設定変更や抽選操作の追跡性を高め、トラブル時の原因調査を可能にする。
   - **内容**:
     - 設定変更、Giveaway作成、終了、再抽選をイベントとして保存
     - 実行者、対象サーバー、変更前後の差分、実行時刻を記録
     - ダッシュボードから期間指定で検索・閲覧可能にする

2. **権限エラー時の理由表示強化**
   - **目的**: ユーザーが「なぜ操作できないか」を即時理解できるようにし、運営問い合わせを減らす。
   - **内容**:
     - サーバー選択不可・設定保存不可・作成不可をケースごとに文言出し分け
     - 必要ロール不足、対象チャンネル未設定、オーナー権限のみなどの理由を表示
     - 解決手順（どこを設定すればよいか）への導線を追加

3. **Giveawayテンプレート機能**
   - **目的**: 定型イベントの作成工数を削減し、記入漏れを防止する。
   - **内容**:
     - タイトル、説明、当選人数、受け取り期限、投稿チャンネルをテンプレート化
     - テンプレートの作成・編集・複製・削除
     - サーバー単位で複数テンプレートを管理し、作成画面でワンクリック適用

4. **下書き保存機能**
   - **目的**: 入力中断時の再入力負担をなくし、投稿ミスを減らす。
   - **内容**:
     - 作成フォーム内容を自動保存（手動保存も可）
     - 期限付き下書き管理（古い下書きの自動クリーンアップ）
     - 再訪時に「復元する / 破棄する」を選べるUIを提供

5. **失敗時リトライ/通知改善**
   - **目的**: 一時的な通信・API障害時の操作失敗を最小化する。
   - **内容**:
     - 一時エラー時の自動リトライ（上限回数・待機時間つき）
     - 失敗時の詳細メッセージ（エラー種別、再試行可否）
     - ユーザー操作可能な「再試行」ボタンと状態表示（処理中/完了/失敗）

6. **重複作成防止（冪等性キー）**
   - **目的**: 連打や再送による同一Giveawayの重複投稿を防ぐ。
   - **内容**:
     - 作成リクエストごとに冪等性キーを発行・検証
     - 同一キーの再送時は新規作成せず既存結果を返す
     - UI側で送信ボタンの二重押下防止と処理状態の固定を実装

7. **入力値バリデーションの可視化強化**
   - **目的**: 入力エラーを早期に検知し、保存/作成時の失敗率を下げる。
   - **内容**:
     - 当選人数、期限、タイトル文字数のリアルタイム検証
     - フィールド単位でエラー理由と修正例を表示
     - サーバー設定（許可チャンネル・ロール）との整合チェックを事前実行

## 優先度: 中（運用効率の改善）

8. **Giveaway複製機能**
   - **目的**: 類似キャンペーンを短時間で再実施可能にする。
   - **内容**:
     - 既存Giveawayの設定を引き継いで新規作成
     - 日時や当選人数だけ差し替えて再利用
     - 複製元の参照リンクを履歴に保持

9. **フィルター/検索の拡張**
   - **目的**: 過去開催の確認を迅速化し、運用判断をしやすくする。
   - **内容**:
     - 作成者、チャンネル、期間、ステータス、キーワード検索を追加
     - 複合条件検索と並び替え（新着順、終了順、参加者数順）
     - よく使う条件を保存してワンクリック再利用

10. **参加・当選統計ダッシュボード**
    - **目的**: Giveaway施策の成果を可視化し、改善サイクルを回せるようにする。
    - **内容**:
      - 開催回数、参加者数推移、当選分布、受け取り率を表示
      - 期間比較（今週/先週、今月/先月）を提供
      - サーバー全体とチャンネル別の集計を切り替え可能にする

11. **通知チャンネル設定**
    - **目的**: 運営への情報連携を自動化し、見逃しを減らす。
    - **内容**:
      - 終了通知、再抽選通知、受け取り期限切れ通知の送信先を分離設定
      - 通知テンプレート（簡易文言）をカスタマイズ可能にする
      - テスト送信で設定内容を事前確認できるようにする

12. **ロール設定プリセット**
    - **目的**: 新規導入時や権限見直し時の設定工数を削減する。
    - **内容**:
      - 小規模/中規模/大規模向けの初期権限セットを提供
      - 読み取り専用、作成可能、管理者向けなど役割別テンプレートを用意
      - 適用前に差分プレビューを表示し、意図しない上書きを防止

13. **スケジュール予約投稿**
    - **目的**: イベント開始時刻に合わせた自動投稿で運用負荷を下げる。
    - **内容**:
      - 将来日時を指定してGiveawayを予約作成
      - 予約一覧で編集・取消を可能にする
      - 実行失敗時の再試行と通知を提供

14. **ロック/一時停止機能（メンテ運用）**
    - **目的**: サーバーイベント時に新規作成を一時停止し、運用事故を防ぐ。
    - **内容**:
      - サーバー単位で作成機能を一時停止するトグル
      - 既存Giveawayのみ管理可能な制限モード
      - 停止中に表示する案内文をカスタマイズ

## 優先度: 低（将来的な拡張）

15. **多言語文言の強化（i18n拡張）**
    - **目的**: 日本語/英語運用で同品質の体験を提供する。
    - **内容**:
      - エラーメッセージ、通知、管理UI補助文言の翻訳追加
      - 言語ごとの文言差分テストを導入
      - 管理者向けに既定言語設定を追加

16. **Webhook連携**
    - **目的**: 外部運用基盤（管理ダッシュボード、通知サービス）との連携を容易にする。
    - **内容**:
      - 作成/終了/再抽選などのイベントをWebhook送信
      - 署名付きリクエストと再送制御を実装
      - イベント種別ごとの購読設定を提供

17. **CSVエクスポート**
    - **目的**: 監査・レポート作成・手動分析をしやすくする。
    - **内容**:
      - Giveaway一覧、当選結果、監査ログをCSVで出力
      - 期間、ステータス、チャンネルで抽出条件を指定
      - 個人情報の出力範囲を権限別に制御

18. **ヘルプ/オンボーディング導線**
    - **目的**: 初回利用者の学習コストを下げ、導入定着率を上げる。
    - **内容**:
      - 初回ログイン時のセットアップチェックリスト表示
      - 各画面にチュートリアル導線とFAQリンクを配置
      - 設定未完了時に不足項目を案内するガイドを表示

19. **テーマ/表示カスタマイズ**
    - **目的**: 運用者の見やすさを向上し、長時間運用の負担を減らす。
    - **内容**:
      - ダークモード、文字サイズ調整、テーブル密度切替
      - 一覧表示列の表示/非表示設定
      - 個人設定をアカウント単位で保持

20. **API利用メトリクス可視化**
    - **目的**: 性能劣化や障害の予兆を早期把握する。
    - **内容**:
      - API成功率、平均応答時間、失敗率を可視化
      - 失敗率上昇時のアラート閾値を設定
      - 管理画面で直近24時間/7日間を確認可能にする

3. 複製機能 + 検索拡張 + 予約投稿  
4. 統計ダッシュボード + 通知設定 + ロールプリセット  
5. i18n / Webhook / CSV / UX拡張（オンボーディング・テーマ）
