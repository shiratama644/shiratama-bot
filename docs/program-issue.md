# applejp-bot プログラム問題一覧

このドキュメントは、現行実装を探索して見つかった「セキュリティ以外」のプログラム上の問題を整理したものです。  
（確認日: 2026-05-16）

## 1) 受け取り期限フォーマットの仕様不一致（`w` が実装で未対応）

- **現象**
  - 文言・プレースホルダでは `1w`（週）を受け取り期限として使えるように見えるが、実際の期限計算は `m/h/d` のみ対応。
  - `1w` を設定した場合、終了後の受取期限表示・判定が期待どおりにならない。
- **原因**
  - 受け取り期限の計算関数 `parseDurationSeconds` が `^(\d+)(m|h|d)$` しか受け付けていない。
- **影響**
  - UI/文言と実挙動がずれ、運用者が設定ミスに気づきにくい。
- **該当箇所**
  - `backend/src/i18n.ts`
  - `backend/src/giveaway/embed.ts`
  - `backend/src/giveaway/service.ts`

## 2) `/gsettings` でロール・チャンネル設定を空に戻せない

- **現象**
  - `/gsettings` の UI ではロール/チャンネル選択に `minValues=1` が設定されており、未設定（空配列）に戻せない。
- **原因**
  - コマンド UI 側の入力制約と、DB/API 側の「空配列を許容する設計」が不一致。
- **影響**
  - 一度設定した内容をクリアできず、設定変更の自由度が下がる。
- **該当箇所**
  - `backend/src/commands/gsettings.ts`
  - `backend/src/db/guildSettings.ts`

## 3) 過去日時の締切を作成できる

- **現象**
  - 締切入力がパース可能であれば、過去日時でも作成できる。
- **原因**
  - `parseDeadline` に「未来日時であること」の検証がない。
- **影響**
  - 作成直後に終了対象となる Giveaway が発生し、運用ミスにつながる。
- **該当箇所**
  - `backend/src/deadline.ts`
  - `backend/src/giveaway/service.ts`

## 4) メンバー一覧の取得が欠落しやすい構成

- **現象**
  - `/api/guilds/:guildId/options` のメンバー取得で `guild.members.fetch()` が失敗すると、オーナー1名だけを返すフォールバックになる。
  - 実行時 Intent が不足すると、このフォールバックが発生しやすい。
- **原因**
  - `GuildMembers` Intent を使わずに全メンバー取得を前提とした実装になっている。
- **影響**
  - ダッシュボード上のメンバー情報が不足し、再抽選結果表示などで `Unknown user` が増える。
- **該当箇所**
  - `backend/src/bot.ts`
  - `backend/src/api.ts`
